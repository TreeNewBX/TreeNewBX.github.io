初试 Flutter

好好的怎么跑来做 flutter 开发了，好吧，其实没啥原因，无非是公司需要，只能指哪打哪了:)
幸好自己也对移动端开发有点兴趣，也想折腾下移动开发，毕竟它前几年那么火，现在一下子能开发两个平台的 app，那当然是火上浇油，干柴烈火了

使用 flutter 的第一步就是得补补面向对象的知识了，之前做 web 类开发其实很少使用到面向对象的知识，或者说 JS 这门语言面向对象味没那么多,

而到了 flutter 中使用 dart 进行开发后大部分代码都是用类组织起来的，代码被类分割成一块一块，其实和 react 的开发体验比较像，但可惜我没用 react 做过正式项目，很多方面还不熟悉。

再一个就是适应 flutter 大量的嵌套，将样式和逻辑混合固然可以使得代码得到很好的分割，当你一旦想做出点复杂效果时就必然会写出大量嵌套的代码，不得不说这些代码看着也挺费劲的，幸好有开发工具的辅助，要不然光嵌套就能把人搞死。

但 flutter 的高性能还是挺舒服的，之前写小程序时，项目越来越大，性能到后面就有点差了（当然这也不怪小程序，毕竟人家的定位是“小”，是我们非得写大的），

后面切换到 flutter 后就很舒服了，性能问题完全不用担心，媲美原生。

flutter 最近的更新比较有意思的点还有对桌面端的支持，我周末试了下，操作蛮简单的，至少比当初尝试 electron 时方便很多，等后面成熟起来相信也是开发桌面端的一把好手。

关于上手难度感觉还是不大的，毕竟 dart 本来就是冲着替代 js 去的，也符合 ECMAScript 标准，其实跟 JS 是近亲，对前端来说上手成本很低（而相对来说 Mozilla 推出的 Rust 语法我是真的看不懂很多...)，

之后就是熟悉 flutter 的组件开始干活了，其中大量的组件在 web 中都有对应，感觉就是把 css 那一堆拿过来重新写了一遍，上手难度也不大。

如果有小伙伴想上手的可以大胆尝试，不会后悔的。
